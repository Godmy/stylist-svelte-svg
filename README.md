# Stylist-Svelte

`stylist-svelte` is a Svelte 5 component library with TypeScript typings, theme support, and a large generated component surface organized by domain.

## Installation

```bash
npm install stylist-svelte svelte
```

Peer dependency:

- `svelte@^5`

## Quick Start

```svelte
<script lang="ts">
	import { ThemeProvider, Button } from 'stylist-svelte';
</script>

<ThemeProvider initialMode="light" initialScheme="minimal">
	<Button variant="default">Get started</Button>
</ThemeProvider>
```

## Theme Model

The theme system is split into two axes:

- `mode`: `default`, `light`, `dark`
- `scheme`: `minimal`, `ocean`, `forest`, `sunset`

`ThemeProvider` accepts:

- `initialMode?: TokenThemeMode`
- `initialScheme?: TokenThemeScheme`
- standard `div` attributes
- `children`

## Package Surface

- Primary entry point: `stylist-svelte`
- Types are published from `dist/index.d.ts`
- Svelte runtime entry is published from `dist/index.js`
- The package currently exports the root entry point only
- Story files, demo files, tests, and generated JSON metadata are not part of the public npm package

## Repository Layout

The source code is centered around `src/lib` and grouped by domain:

- `control`, `input`, `layout`, `navigation`, `theme`, `typography`
- `animation`, `canvas`, `chart`, `media`, `notification`
- `chat`, `commerce`, `file`, `form`, `geo`, `management`, `science`, and others

Within each domain, files are usually separated into:

- `component`
- `function`
- `class`
- `type`
- `interface`
- `const`

Many `index.ts` barrels are generated by the existing indexation workflow.

## Development

Useful commands:

```bash
yarn dev
yarn build
yarn validate
yarn test:unit
```

Build notes:

- `yarn build` runs SVG data generation before `svelte-package`
- `yarn validate` runs linting, TypeScript, and `svelte-check`
- generated `src/lib/**/index.ts` files are managed by the existing scripts in `scripts/`
