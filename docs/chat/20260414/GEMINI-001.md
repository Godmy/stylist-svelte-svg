# Архитектурная сессия: 2026-04-14
## Тема: Методология SAMO и развитие stylist-svelte

### GEMINI-001-answer-to-USER-001

**1. Улучшение и расширение состава `<cluster>\<joint>`**
Методология SAMO имеет сильный фундамент. Для расширения до «мирового уровня» предложено:
* Добавить кластер `meta` для метаданных (SEO, манифесты).
* Добавить кластер `logic` для сложных бизнес-правил, выходящих за рамки простых функций.

**2. Константы домена: `<const>\<enum>\index.ts`**
Поддерживаю создание реестра констант.
* `<const>\<enum>\<family>\index.ts` — для группировки (например, `dimensions`, `timing`).
* Для числовых констант типа `DEFAULT_WIDTH` лучше использовать семейство `default` или `limit`.

**3. Переименование `enum` в `list`**
Переименование `<const>\<enum>` в `<const>\<list>` и `<type>\<enum>` в `<type>\<list>` сделает архитектуру менее привязанной к синтаксису конкретного языка (TypeScript `enum` не всегда подходит для `as const` объектов).

**4. Новый joint: `configuration`**
Ввести `<const>\<configuration>\<family>` для сложных объектов настроек. Это разгрузит `struct`.

**5. Новый joint: `single`**
Для констант и типов, существующих в единственном экземпляре (синглтонов на уровне данных).

**6. Новый joint: `factory` или `compose`**
`interface/factory` — более узнаваемый паттерн (Factory Method/Abstract Factory). `compose` подходит для паттерна **Composition**, что актуально для Svelte 5.

**7. Паттерн в `interaction/interface/proto/clickable`**
Это паттерн **Trait** (или **Mixin Interface**). Обеспечивает **Interfacability** (интерфейсную совместимость) через «прото-сущности».

**8. Другие паттерны в проекте**
* **Manager** (`class/manager`): Repository/Controller.
* **State Machine** (`function/state`): Реактивное состояние Runes.
* **Atomic Design**: Atoms/Molecules/Organisms.

**9. Кластер `function/method/family`**
Хорошее название для функций, которые действуют как "методы расширения" над данными домена.
