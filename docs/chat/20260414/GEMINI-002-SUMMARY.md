## Сводная таблица предложений и консенсуса

| Пункт вопроса | Предложение / Консенсус | Статус | Основные аргументы |
| :--- | :--- | :--- | :--- |
| **1. Состав `<cluster>\<joint>`** | Легализовать `interface/component`, `interface/contract`, `interface/record`, `class/state-manager`, `const/icon`. | **Консенсус** | Фактическое дерево проекта (более 500 файлов `component`) требует признания в ADR/AGENTS. |
| **2. Реестр констант** | Отклонить `index.ts` на весь домен (God-file). Использовать существующие семейства + автобаррели. | **Консенсус** | Сохранение SRP и предотвращение конфликтов импорта. |
| **3. Переименование `enum` → `list`** | **Отклонить.** Оставить `enum`. | **Дискуссия** | `enum` — семантика закрытого словаря. Высокая стоимость миграции (~2000 файлов). |
| **4. Новый joint `configuration`** | Добавить `const/configuration` и `type/configuration`. | **Консенсус** | Чёткое отделение настроек/параметров от структур данных (`struct`). |
| **5. Новый joint `single`** | Предложено заменить на **`value`** или **`alias`** (для типов). | **Дискуссия** | `single` описывает количество, а не роль. `value` семантически точнее. |
| **6. Паттерн `factory`** | Ввести `interface/factory` для DTO создателей. | **Консенсус** | Разделение: `recipe` — чертёж состава, `factory` — входные данные для создания. |
| **7. Паттерн `proto`** | Назван **Role Interface** или **Capability Interface**. | **Консенсус** | Атомарное поведение (ISP из SOLID). Патентопригодный термин. |
| **8. Другие паттерны** | Выделена иерархия: `proto` → `recipe` → `component`. | **Инсайт** | Ядро методологии SAMO, уникальное для проекта. |
| **9. Кластер `method`** | **Отклонить.** Использовать `script`, `transform` или `logic`. | **Консенсус** | Избежание путаницы с ООП-терминологией в функциональной среде. |

## Предложенные решения для следующего шага:

1. **Легализация фактической структуры**: Внести `interface/component` и `interface/record` в регламент.
2. **Формализация иерархии**: `proto` (Role) → `recipe` (Composite) → `component` (Facade).
3. **Расширение joints**: Ввести `configuration` и `value`.
4. **Отказ от миграции `enum`**: Сохранение текущего именования для стабильности.
5. **Легализация `hook`**: Для паттернов Svelte 5 Runes.
