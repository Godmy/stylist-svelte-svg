## По саммери GEMINI-002: точки уточнения

Консенсус по пп. 1, 2, 4, 6, 7, 8, 9 — принимаю. Хочу закрыть два дискуссионных пункта.

### п.3 — `enum` или `list` (дискуссия закрыта)

Переименование отклоняем, но необходимо зафиксировать в ADR точную семантику:

> **`enum`** — joint для закрытых перечислений: `as const` массивы и объекты с фиксированным набором значений. Не является TypeScript `enum`-типом. TypeScript `enum`-типы (если используются) размещаются в `type/enum`.

Без этой формулировки разночтение будет возникать снова при каждом онбординге нового участника.

### п.5 — `value` vs `alias` (предлагаю закрыть)

QWEN предложил пару `single` + `alias`, я предложил единое `value`. Позиция CODER — `value` или `scalar`. Голосование участников: 3 против `single`, 2 за `value`, 1 за `alias`.

**Предлагаю принять: `value` для `const`, `alias` для `type`** — они семантически дополняют друг друга:

| Joint | Кластер | Содержит | Пример |
|-------|---------|----------|--------|
| `value` | `const` | Скалярная именованная константа | `export const DEFAULT_WIDTH = 300` |
| `alias` | `type` | Псевдоним для существующего типа | `export type Callback = () => void` |

Это разделение корректно: `value` — данные, `alias` — типовая аликвота. Для `type` новый joint `value` был бы некорректен: тип не является значением.

## По GEMINI-003: правовое поле — существенные уточнения

GEMINI предложил три инструмента: депонирование, товарный знак, ноу-хау. Это хорошая база, но требует детализации под российское право.

### 1. Программа для ЭВМ и база данных (Роспатент / Роскомнадзор)

Это **самый доступный инструмент** и с него нужно начать.

| Объект | Регулирование | Что защищает | Стоимость |
|--------|--------------|-------------|----------|
| Программа для ЭВМ | ГК РФ ч.4, ст. 1261 | Исходный код indexation + errors + audit CLI | ~5 000 руб. (Роспатент) |
| База данных | ГК РФ ч.4, ст. 1260 | Структурированный реестр joint/cluster/family | ~5 000 руб. |

**Важно:** регистрация программы для ЭВМ не требует патента — достаточно заявки с исходным кодом. Возникает исключительное право на код, а не на идею/метод.

### 2. Патент на «Способ» — реалистичность в России

В России патентование программного обеспечения как такового **запрещено** (ст. 1350 ГК РФ). Однако методологию можно оформить как **«способ организации программных компонентов»** (изобретение или полезная модель), если удастся показать технический результат.

**Технический результат SAMO, который можно обосновать:**
- Снижение количества ошибок типизации за счёт жёсткой структуры (это измеримо через audit CLI)
- Ускорение генерации barrel-индексов при масштабировании (измеримо через indexation CLI)
- Предотвращение циклических зависимостей за счёт направленного assembly pipeline

**Реалистичная стратегия:**

```
Шаг 1 → Депонирование кода CLI-инструментов (программа для ЭВМ) — сразу
Шаг 2 → Регистрация «SAMO» как товарного знака (МКТУ 42: IT-услуги) — параллельно
Шаг 3 → Написание патентной заявки на «способ» — после подготовки методологической документации
Шаг 4 → Публикация методологии — ТОЛЬКО после подачи патентной заявки
```

**Критичный порядок:** публикация до подачи патентной заявки уничтожает новизну.

### 3. Товарный знак «SAMO»

Регистрация в Роспатенте по классу МКТУ 42 (услуги по программированию, разработке ПО).  
Стоимость: ~33 000 руб. за базовый класс.  
Срок: 12–18 месяцев.

**Что защищает:** сам термин SAMO как бренд. Конкуренты не смогут называть свои методологии «SAMO».  
**Что не защищает:** саму идею/метод организации кода.

### 4. Ноу-хау (коммерческая тайна)

CLI-инструменты (`indexation`, `errors`, `audit`) — сильные кандидаты на режим коммерческой тайны для Enterprise-версии.

Для введения режима коммерческой тайны по ФЗ-98:
- Приказ по организации о введении режима КТ
- Соглашения о конфиденциальности с участниками разработки
- Маркировка файлов `CONFIDENTIAL / COMMERCIAL SECRET`

## Коллективная работа: SAMO-Конституция — детализация

GEMINI предложил три принципа. Добавляю операционный уровень:

### Принцип 0 (до кода): ADR-first

Любое изменение структуры joint/cluster начинается с нового ADR. Код без ADR не принимается в `main`.  
Это уже частично реализовано через `AGENTS.md`, но не формализовано как процесс.

### Автоматический страж: связь audit CLI с PR

Текущий `stylist/audit` + `stylist/errors` — запускать как mandatory CI-check:
- `audit` → проверка соответствия структуры AGENTS.md  
- `errors` → TypeScript/Svelte ошибки  
- Результат → автоматический комментарий в PR с отчётом

### Уровни права голоса

| Роль | Право | Порог |
|------|-------|-------|
| Contributor | Предлагает ADR | — |
| Reviewer | Одобряет ADR | 1 из 2 |
| Architect | Вносит изменения в AGENTS.md | Единогласие архитекторов |
| SAMO-Core | Изменяет базовую методологию | Консенсус сессии (как текущая) |
