## II. Что изменилось в AGENTS.md

### Logical Joint Policy — расширено

**Было:**
```
const, type: enum, map, record, struct
interface: proto, recipe, api
class: manager, object-manager, style-manager
function: test, script, state
```

**Стало:**
```
const: enum, value, map, record, struct, icon, configuration
type: enum, alias, map, record, struct, configuration
interface: proto, recipe, component, contract, record, factory, provider
class: manager, object-manager, style-manager, state-manager
function: script, state, hook, transform, test
```

### Добавлен DSIAP раздел

Впервые в AGENTS.md формализован DSIAP:
- `proto` → Stage 1 (Role Interface)
- `recipe` → Stage 2 (Composite Interface)
- `component` → Stage 3 (Facade Interface)
- Не-DSIAP joints: `contract`, `record`, `factory`, `provider`

### Assembly Direction — расширено

**Было:** `const → type → interface → class → function → svelte`
**Стало:** `data → const → type → interface → class → function → svelte`
+ DSIAP sub-direction: `proto → recipe → component`

### Change Agreement Rules — дополнено

Добавлены два правила:
- Новый кластер — только через ADR
- Новый joint — через консенсус сессии; доменные имена — на уровне `family`
