# Справочник архитектуры SAMO: Кластеры и Соединения
**Дата:** 2026-04-14
**Документ:** GEMINI-005-REFERENCE (Полный глоссарий)

---

## I. КЛАСТЕРЫ (Clusters)
Кластер определяет тип языка программирования или формат файла.

| Кластер | Описание |
| :--- | :--- |
| **`const`** | Константы — неизменяемые данные. |
| **`type`** | Описания типов TypeScript. |
| **`interface`** | Контракты и интерфейсы (структурная сборка). |
| **`class`** | Сущности с состоянием и методами. |
| **`function`** | Чистая логика и процедуры. |
| **`svelte`** | Svelte-компоненты (UI). |
| **`data`** | Статические ресурсы (JSON, SVG, MD). |

---

## II. СОЕДИНЕНИЯ (Joints)

### 1. Кластеры `const` и `type`
| Joint | Назначение |
| :--- | :--- |
| **`enum`** | Закрытые словари (`as const`). |
| **`value`** | Одиночные скаляры (напр. `DEFAULT_WIDTH`). |
| **`struct`** | Сложные объекты (DTO). |
| **`record`** | Таблицы подстановки (Lookup). |
| **`map`** | Сложные мапы. |
| **`alias`** | Псевдонимы типов (только для `type`). |
| **`config`** | Настройки поведения системы. |

### 2. Кластер `interface` (DSIAP - Contract Model)
| Joint | Роль (Паттерн) | Описание |
| :--- | :--- | :--- |
| **`proto`** | **Role Interface** | Атомарная способность. |
| **`recipe`** | **Composite Interface** | Архитектурный чертеж компонента. Сборка через **Static Assembler**. |
| **`contract`** | **Facade / Bound Interface** | Финальное обязательство (включая темы/контексты). |
| **`factory`** | **DTO Interface** | Параметры для функций-создателей. |
| **`provider`** | **Context Interface** | Контракты для Svelte Context. |

### 3. Кластер `class` (Управление)
| Joint | Роль |
| :--- | :--- |
| **`director`** | Высшее управление ресурсами. |
| **`builder`** | Архитектурная сборка сложных объектов. |
| **`manager`** | Операционный контроллер логики. |
| **`object-manager`** | Репозиторий однотипных объектов. |
| **`style-manager`** | Декоратор визуальной логики. |

### 4. Кластер `function` (Логика)
| Joint | Тип функции |
| :--- | :--- |
| **`script`** | Утилиты и процедуры. |
| **`state`** | Порождение реактивного состояния. |
| **`hook`** | Реактивные паттерны и жизненный цикл. |
| **`transform`** | Чистые преобразования данных. |
| **`logic`** | Доменные бизнес-алгоритмы. |
| **`test`** | Сценарии верификации. |

---

## III. Центральный паттерн сборки
**`RecordArchitectureMerge`** — инструмент реализации паттерна **Static Assembler (Type Builder)**. Позволяет синтезировать финальный интерфейс на основе массива спецификаций, обеспечивая морфологическую целостность домена.
