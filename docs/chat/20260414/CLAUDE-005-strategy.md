### Задача 1.1: Создать ADR-005

Документ в 6 разделах (по CODER-002):

| Раздел | Содержание |
|--------|-----------|
| Проблема | Расхождение регламента и живого дерева |
| Термины | Обновлённый словарь joint (финал сессии) |
| Кластеры | Полный обязательный перечень с ограничениями |
| Joints | Разрешённые joints по каждому кластеру |
| DSIAP | Паттерн `proto → recipe → contract` — формальное определение |
| Миграция | Что легализуем, что мигрируем, что запрещаем |

Финальный словарь joints (итог голосования):

| Кластер | Joints (было) | Joints (стало) |
|---------|--------------|----------------|
| `const` | `enum`, `map`, `record`, `struct` | + `value`, `configuration` |
| `type` | `enum`, `map`, `record`, `struct` | + `alias`, `configuration` |
| `interface` | `proto`, `recipe`, `api` | + `contract` (→ основной фасад), `record`, `provider`; `api` — под вопросом |
| `class` | `manager`, `object-manager`, `style-manager` | + `state-manager` |
| `function` | `script`, `state` | + `hook`, `transform` |
| `svelte` | `atom`, `molecule`, `organism` | без изменений |
| `data` | `json`, `shader`, `svg` | + `md` (легализовать) |

Отклонено: `class/director`, `class/builder`, `function/logic`, `function/method`, переименование `enum → list`.

## Блок II. Правовая защита — строгая последовательность

Сессия единогласно определила порядок. Он не обсуждается — он исполняется.

### Шаг 2.1: Зафиксировать авторство и даты (немедленно, бесплатно)

- Убедиться, что git-история сессии и репозиториев ведётся непрерывно
- Git commit date = доказательство даты создания
- Для дополнительной страховки: отправить ключевые документы (ADR.md, AGENTS.md, GEMINI-006 в редакции) на email себе с отметкой времени или загрузить в нотариально заверенное облако

Срок: эта неделя. Стоимость: 0 руб.

### Шаг 2.2: Депонирование текстов (до публикации)

- ADR.md, AGENTS.md, QWEN.md, GEMINI-005-GLOSSARY (финальная редакция)
- Регистратор: Роспатент или РАО (Российское авторское общество)
- Стоимость: ~2 000–5 000 руб. за пакет документов
- Срок: 1–2 месяца

**Не путать с патентом.** Депонирование фиксирует дату создания текста, а не защищает идею.

### Шаг 2.3: Регистрация CLI как "Программы для ЭВМ" (до публикации)

- Объекты: `stylist/indexation`, `stylist/errors`, `stylist/audit` (закрытый репозиторий)
- Регулирование: ГК РФ ст. 1261
- Стоимость: ~3 000–5 000 руб. за каждую программу
- Срок: 1–2 месяца
- Подаётся в Роспатент, не требует экспертизы

**Это самый важный шаг.** Код CLI — единственный актив, защищаемый строго и дёшево.

### Шаг 2.4: Регистрация товарного знака "SAMO"

- До подачи: провести предварительный поиск по базе Роспатента (ФИПС) на конфликты
- Класс МКТУ 42 (разработка ПО, IT-услуги) + возможно 35 (консалтинг)
- Стоимость: ~33 000–40 000 руб.
- Срок: 12–18 месяцев
- Параллельно: зарегистрировать домены samo.dev, samo.ru, stylist.dev

Срок старта: после предварительного поиска (можно делать параллельно с шагом 2.3).

### Шаг 2.5: Выбор лицензии для публичного кода

Решение до открытия репозитория. Рекомендация:

| Лицензия | Плюсы | Минусы | Для SAMO |
|----------|-------|--------|----------|
| MIT | Максимальный adoption | Нет патентной защиты | Слабовата |
| Apache 2.0 | Явная патентная оговорка | Чуть сложнее | **Рекомендую** |
| BUSL (Business Source License) | Коммерческий контроль | Меньше adoption | Рассмотреть для tooling |

Итог: **Apache 2.0 для библиотеки + BUSL или проприетарная для CLI-инструментов.**

### Шаг 2.6: Патентная стратегия — только исследование

Патент сейчас — не инструмент, а ресурс-поглотитель. Требует:

1. **Prior art review** — формальный обзор аналогов: FSD, Atomic Design, Clean Architecture, DDD foldering, contract-first UI systems. CODER-006 сделал его черновик — нужна финальная версия с юристом.
2. **Claim engineering** — формулировка формулы изобретения с патентным поверенным. Самостоятельно — риск отклонения.
3. **Enablement package** — воспроизводимые метрики технического результата.

Стоимость подготовки + подачи: 150 000–250 000 руб. Срок рассмотрения: 18–24 месяца.

**Вердикт:** начинать только после шагов 2.1–2.5 и после появления первых коммерческих результатов, подтверждающих ценность методологии.

## Блок III. Коммерциализация — последовательность, не параллелизм

Ключевой вывод CODER-004: **consultancy first, tooling second, platform later.**

Сессия единогласна: текущая фаза — это распространение, не монетизация.

### Этап 3.1: Публикация и adoption (0–6 месяцев)

**Цель:** стать известными в Svelte и design system community.

Действия:
- Открыть `stylist-svelte` с лицензией Apache 2.0
- Опубликовать ADR.md, AGENTS.md, QWEN.md как методологическую документацию
- Написать 3–5 технических статей: "Как мы организуем 28 доменов", "DSIAP: трёхуровневая сборка интерфейсов", "Автоматический аудит архитектуры"
- Сделать демо-страницу с интерактивной картой доменов (используя уже готовый `stylist-svelte/src/routes`)
- Выступить на Svelte Society, ViteConf, или локальных митапах

**Не делать на этом этапе:** эквайринг, платные тиры, регистрацию компании.

### Этап 3.2: Первый платный продукт — архитектурный аудит (3–9 месяцев)

**Цель:** первые деньги и измеримые кейсы.

Оффер:
> "Архитектурный аудит вашего Svelte/TS design system. За 5 рабочих дней вы получаете: карту нарушений структуры, сравнение с SAMO-стандартом, план миграции и CI-правила для предотвращения деградации."

Почему это работает:
- Не требует продуктовой инфраструктуры
- Быстро валидирует спрос
- Даёт кейсы с измеримыми результатами (до/после)
- Создаёт материал для следующего продукта

Канал продаж: прямой outreach в design system команды крупных компаний, LinkedIn, конференции.

ICP (по CODER-004): Design System Lead в продуктовой компании с командой 10–50 разработчиков.

### Этап 3.3: Enterprise tooling (6–18 месяцев)

После накопления 3–5 кейсов аудита:

- Упаковать `audit` CLI в Enterprise Policy Pack (закрытая версия с расширенными правилами)
- Добавить CI/CD интеграции (GitHub Actions, GitLab CI)
- Добавить executive-level отчёты
- Лицензия: годовая на организацию

**SAMO-Lint (ключевой приоритет):** плагин для VS Code и JetBrains, проверяющий структуру в реальном времени. Это то, за что платит каждый разработчик ежедневно. Это конкурентный ров, который тяжело скопировать.

### Этап 3.4: Сайт с эквайрингом (12–24 месяца)

Только после появления продуктов с повторяемыми продажами.

Минимальная конфигурация:
- SvelteKit (нативно для проекта)
- ЮKassa для РФ (ИП или ООО, 54-ФЗ касса)
- Stripe для международных клиентов (через нерезидента)
- Контент: landing, документация, playground, pricing

**Не открывать сайт с эквайрингом до:** первых повторяемых продаж и оформленного юридического лица.

## Блок IV. Что делать с GEMINI-005 и GEMINI-006

### GEMINI-005-GLOSSARY

Разделить на две части:

**Нормативная часть** (в ADR-005):
- Все joints, по которым достигнут консенсус сессии
- Без `director`, `builder`, `logic`
- `contract` как третий уровень DSIAP — ✅ принято

**Предложения второй волны** (отдельный DRAFT-GLOSSARY-v2):
- `class/director`, `class/builder` — требуют живых примеров
- `function/logic` — требует разграничения с `script`
- `interface/api` — 5 файлов в `input/`, требует формального определения

### GEMINI-006-PATENT-BASE

Документ полезен как внутренний манифест. Для использования в правовых целях требует:

1. Убрать неизмеримые заявления ("нулевая избыточность", "гарантированная предсказуемость")
2. Добавить раздел "Предшествующий уровень техники" с честным сравнением (черновик в CODER-006)
3. Переформулировать технический результат через измеримые метрики:
   - "Снижение доли нарушений `<cluster>/<joint>` правил при статическом анализе с X% до 0% в CI"
   - "Автоматическое обнаружение 100% структурных нарушений без ручного ревью"
4. Добавить формулу изобретения в виде claim-структуры
5. Работать только с патентным поверенным — не самостоятельно

## Главный вывод сессии

SAMO — это реально сильная архитектурная система. Её ценность сейчас в трёх вещах:

1. **Методология** (ADR + DSIAP) — маркетинговый актив, бесплатный драйвер adoption
2. **CLI-инструменты** (закрытый репо) — коммерческий актив, защищаемый как Программа для ЭВМ
3. **Экспертиза** (аудит + внедрение) — первый бизнес, не требующий инфраструктуры

Всё остальное — следствие того, как хорошо будут работать эти три.
