# GEMINI-001: Подготовка к демонстрации stylist-svelte и stylist

## Дата: 2026-04-22
## Тема: План демонстрации библиотек

### Контекст
Завтра утром (2026-04-23) планируется демонстрация библиотек `stylist-svelte` и `stylist`. Необходимо определить наиболее выигрышные аспекты для показа.

### План анализа
1. Проверить текущую структуру `stylist-svelte/src/lib` на соответствие ADR.
2. Найти примеры наиболее полных "LEGO-компонентов".
3. Оценить состояние `stylist` (логика индексации, генерация баррелей).
4. Проверить наличие `story` файлов для визуальной демонстрации.

## Рекомендации к демонстрации (2026-04-23) — Смена фокуса: "Инфраструктура выше Ассетов"

### 1. Концепция "Инженерного Завода"
* **Месседж:** Мы показываем не библиотеку компонентов (она в альфе), а **платформу для их производства**. 
* **Акцент:** Показать, как автоматизация (`stylist` на Python) позволяет одному-двум разработчикам управлять тысячами файлов без потери контроля.

### 2. Архитектура как Страховка (ADR)
* **Месседж:** "Мы не строим карточный домик". 
* **Что показать:** Строгое следование ADR. Разделение на домены и кластеры. 
* **Почему это важно:** Это гарантия того, что библиотека не превратится в legacy еще до релиза. Любой компонент можно заменить или обновить, не ломая остальные.

### 3. Stylist Engine: Автоматический Аудит (Killer Feature)
* **Месседж:** "Мы управляем качеством через код".
* **Что показать:** Запуск `python stylist/audit/cli.py`.
* **На чем акцентировать внимание:**
    - **6000+ файлов** проверяются за 10 секунд.
    - Автоматический поиск захардкоженных цветов в иконках.
    - Контроль экспортов и соответствия ADR.
* **Результат:** Демонстрация того, что проект — это не куча файлов, а **самодиагностируемая система**.

### 4. Playground: Лаборатория разработки (Status: RECOVERED)
* **Месседж:** Это не витрина для пользователей, а **workbench для инженеров**.
* **Технический статус:** Playground восстановлен после сбоя зависимостей. Для запуска использовать `yarn dev` в папке `stylist-playground`.
* **Что показать:** 
    - Диагностику ошибок.
    - Изолированную среду для каждого из 3000+ сценариев.
    - Как быстро создается новая "история" для теста бага.

### 5. Путь к Бете (Roadmap)
* **Месседж:** "Каркас готов, масштаб задан, впереди — наполнение логикой".
* **Акцент:** Четкие критерии стабилизации, основанные на автоматических тестах и проверках движка Stylist.
